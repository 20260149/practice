<Week 3 AI 대화 기록 및 실습 기록>
(1) Pygame 첫 실행 및 게임 루프 이해
Q. Pygame으로 800x600 창을 만들고 흰 배경 위에 파란 원을 화면 중앙에 표시해줘.
=> 실행 결과
800x600 크기의 창 생성
흰색 배경 위 중앙에 파란 원 표시
창 닫기 버튼을 누르면 정상 종료됨

Q. 아래 Pygame 코드를 처음 프로그래밍하는 사람에게 설명해줘. 각 줄이 무엇을 하는지, 왜 필요한지 알려줘.
=> 실행 결과
pygame.init() : Pygame 라이브러리 초기화
set_mode() : 게임 창 생성
set_caption() : 창 제목 설정
screen.fill() : 이전 프레임 화면 지우기
draw.circle() : 화면에 원 그리기
display.flip() : 현재 프레임을 화면에 반영
clock.tick(60) : 초당 60번 반복되도록 속도 제한

(2) 코드 수정 실험
Q. 원의 색을 빨간색으로 바꾸고 크기를 100으로 변경해줘.
=> 실행 결과
파란 원 → 빨간 원 변경
반지름 증가로 원 크기 확대

(3) 키보드 입력 구현
Q. 아래 Pygame 코드에 방향키 입력을 추가해줘. 위/아래/좌/우 키를 누르면 원이 5픽셀씩 움직이게 해줘. 현재 코드: [코드 붙여넣기]
=> 실행 결과
방향키 입력으로 원 이동 가능
x좌표와 y좌표가 실시간으로 변경됨

Q. 이 코드에서 키보드 처리 부분만 설명해줘.
=> 실행 결과
pygame.key.get_pressed() : 현재 눌린 키 상태를 가져옴
pygame.K_LEFT : 왼쪽 방향키 감지
x -= 5 : 왼쪽 이동
x += 5 : 오른쪽 이동
y -= 5 : 위 이동
y += 5 : 아래 이동

Q. y -= 5는 왜 위쪽으로 이동하는지 설명해줘.
=> 실행 결과
Pygame 좌표계는 왼쪽 위가 (0,0)
y값이 작아질수록 위쪽으로 이동

(4) 경계 처리 추가
Q. 현재 코드에 경계 처리를 추가해줘. 원이 화면 밖으로 나가지 못하게 해줘.
=> 실행 결과
화면 끝에서 이동 제한
원 중심 좌표가 창 범위 안에서 유지됨

Q. 속도를 5에서 10으로 바꿔줘.
=> 실행 결과
이동 속도 증가
방향키 반응이 더 빠르게 보임

(5) 추가 기능 시도
Q. Shift 키를 누르면 2배 빠르게 움직이게 해줘.
=> 실행 결과
일반 이동 속도 유지
Shift 입력 시 이동 속도 증가

<시행착오 및 해결과정>
-경계 처리 기능을 추가할 때는 원의 중심 좌표만 제한하면 반지름 때문에 일부가 화면 밖으로 나가는 문제가 있었다. 이를 해결하기 위해 원 반지름까지 고려해서 좌표 범위를 제한하도록 수정했다.
<배운 점>
-이번 실습을 통해 Pygame 프로그램은 단순히 코드를 한 번 실행하는 것이 아니라, 게임 루프 안에서 입력 처리, 화면 지우기, 도형 그리기, 화면 갱신이 반복되어야 정상적으로 동작한다는 점을 이해했다.
