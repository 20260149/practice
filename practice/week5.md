# 'Dodger' 게임 기획 및 프로젝트 준비

## 프로젝트 정보
- 게임 제목: Color Dodger
- 장르: 아케이드 피하기 게임 (Dodger)
- 기반 템플릿: 기존 Dodger 게임 구조를 응용하여 색상 규칙 시스템 추가

## 게임 규칙
- 목표:
  떨어지는 오브젝트를 피하거나 획득하면서 최대한 높은 점수를 얻는다.

- 승리 조건:
  제한 없이 게임을 지속하며 점수를 높이는 것이 목표이다.

- 패배 조건:
  빨간색 오브젝트와 충돌하여 목숨이 모두 소진되면 게임 종료된다.

- 조작 방법:
  - ← : 왼쪽 이동
  - → : 오른쪽 이동
  - ↑ : 위 이동
  - ↓ : 아래 이동

## 오브젝트 규칙
- 빨간색 블록:
  충돌 시 목숨 1 감소

- 초록색 블록:
  충돌 시 점수 +5 증가

- 파란색 블록:
  충돌 시 일정 시간 동안 이동 속도 감소

## 차별화 요소
기존 Dodger 게임은 모든 오브젝트를 피하는 단순 구조이지만,  
본 게임은 색상별로 다른 효과를 적용하여 플레이어가 순간적으로 판단하도록 설계하였다.

## 구현 내용
- Pygame을 이용하여 게임 루프 구성
- 랜덤 오브젝트 생성
- 충돌 판정 구현
- 점수 및 생명 표시 HUD 구현
- 게임 오버 화면 출력

## 제작 과정에서 배운 점
- 게임 루프 구조와 프레임 제어 방식을 이해할 수 있었다.
- 객체 충돌 처리와 리스트 관리 방법을 익혔다.
- 단순 기능에 규칙을 추가하면 게임성이 크게 달라질 수 있음을 경험하였다.

# 5주차 실습 기록

## 선택한 템플릿 및 주제
기존 Dodger 게임 템플릿을 바탕으로, 단순히 장애물을 피하는 구조에서 벗어나 색상별로 다른 효과를 가지는 오브젝트를 추가한 Color Dodger 게임을 기획하였다.  
빨간색은 피해야 하고, 초록색은 점수를 얻으며, 파란색은 이동 속도에 영향을 주도록 설정하여 기존 게임과 차별성을 두었다.

## 사용한 AI 프롬프트 (요약)
1. 기존 Dodger 게임과 차별화할 수 있는 새로운 아이디어가 무엇인지 질문하였다.
2. 색상에 따라 다른 효과를 가지는 게임 규칙을 어떻게 구현할 수 있는지 코드 구조를 요청하였다.
3. Thonny에서 실행 가능한 Pygame 코드 형태로 작성해 달라고 요청하였다.
4. 게임 기획서를 Markdown 형식으로 정리하는 방법을 질문하였다.
5. LMS 제출용 문서 형식에 맞게 내용을 정리해 달라고 요청하였다.

## 코드 구조 설명 요청
AI에게 게임 루프 구조, 오브젝트 생성 방식, 충돌 판정 처리 방법을 설명해 달라고 요청하였다.  
특히 리스트 안에 오브젝트를 저장하고 반복문으로 이동시키는 구조를 이해하는 데 도움이 되었다.

## 기획서 구체화 요청
단순 피하기 게임에서 어떤 요소를 추가하면 차별성이 생기는지 질문하였고, 색상별 기능 분리를 통해 판단 요소를 넣는 방향으로 발전시켰다.

## 구현 가능성 확인
현재 학습 수준에서 구현 가능한 범위인지 확인하기 위해 Pygame 기본 문법으로 작성 가능한지 질문하였다.  
그 결과 키보드 이동, 충돌 처리, 점수 표시까지 포함한 구조로 충분히 구현 가능하다는 점을 확인하였다.

## 배운 점
AI를 활용하면 단순히 코드를 받는 것보다 아이디어를 확장하고 구조를 이해하는 데 도움이 된다는 점을 알게 되었다.  
특히 질문을 구체적으로 할수록 원하는 방향의 결과를 얻기 쉬웠다.
